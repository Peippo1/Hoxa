# Hoxa Information Architecture

## Primary Navigation

The app is organised around four top-level sections:

1. `Plan`
2. `Train`
3. `Progress`
4. `Social`

This matches the core product pillars and keeps the navigation clear for a beginner-friendly audience.

## Global Objects

- active goal
- weekly plan
- workout history
- saved workout templates
- friends and follows
- props activity
- availability preferences

## Plan

### Purpose

Help users understand what is scheduled, why it is scheduled, and how it adapts.

### Key screens

- onboarding
- goal selection
- preferences and availability
- adaptive weekly calendar
- plan adjustments
- upcoming session detail

### Core questions answered

- what am I training for
- what should I do this week
- how does this fit around my schedule
- what changed if I missed a session

## Train

### Purpose

Turn the plan into action with clear session detail and a browsable library.

### Key screens

- today view
- session detail
- workout library
- manual log flow
- completed workout summary

### Core questions answered

- what should I do today
- how long will it take
- what equipment do I need
- how do I log what I actually completed

## Progress

### Purpose

Make consistency, balance, and goal movement obvious without overwhelming the user.

### Key screens

- progress overview
- goal milestone summary
- weekly consistency
- modality balance
- recent training trends

### Core questions answered

- am I staying consistent
- am I improving toward my goal
- am I neglecting mobility or recovery

## Social

### Purpose

Create supportive accountability without turning the product into a noisy social network.

### Key screens

- following list
- activity feed
- props interactions
- profile summary

### Core questions answered

- how are my friends doing
- who supported me recently
- who should I encourage today

## Secondary Navigation And Utilities

- profile and settings
- connected apps and devices
- calendar connections
- notification preferences
- privacy and social visibility settings

## MVP Flow

1. user completes onboarding
2. user selects goal and preferred weekly rhythm
3. app generates first adaptive week
4. user opens session detail and completes or logs workout
5. progress updates
6. social feed reflects completion and props
