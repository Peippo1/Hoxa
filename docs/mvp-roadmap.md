# Hoxa MVP Roadmap

## Build Goal

Ship a portfolio-quality MVP that demonstrates a clear product point of view, elegant cross-platform UX, and clean architecture ready for future integrations.

## Phase 0: Foundation

### Objectives

- define product scope and information architecture
- establish visual system and tone
- scaffold iOS, web, and backend repositories
- create shared example plans and sample users

### Deliverables

- root README
- product brief
- architecture documentation
- starter web and iOS shells
- backend contract scaffold

## Phase 1: Core Planning Flow

### Objectives

- support onboarding and goal setup
- generate a starter weekly calendar using mock adaptive rules
- model plan weeks, sessions, completion state, and schedule constraints

### Deliverables

- onboarding flow
- goal selection flow
- availability and preference inputs
- adaptive weekly plan screen
- session detail models

### Notes

This is the highest-priority phase because it proves the product differentiator.

## Phase 2: Training Experience

### Objectives

- surface session detail views
- build workout library taxonomy
- support manual workout logging and completion tracking

### Deliverables

- session detail screen
- workout library
- manual workout logging
- workout completion state
- simple post-session reflection fields

## Phase 3: Progress Layer

### Objectives

- show consistency and trend value quickly
- connect completed sessions to goal progress

### Deliverables

- weekly consistency summary
- monthly totals by modality
- streaks and goal progress
- balance view across run, strength, mobility, and recovery

## Phase 4: Lightweight Social

### Objectives

- add low-pressure accountability features
- support follows and props without a heavy feed product

### Deliverables

- friend list
- recent activity feed
- props interactions
- profile basics

## Phase 5: Integration Architecture

### Objectives

- prepare the platform for future sync work
- validate data boundaries and adapters

### Deliverables

- Apple Health and Apple Watch service protocols
- Garmin import adapter placeholder
- Google Calendar and Apple Calendar adapter placeholder
- ingestion job design
- sync event models

## Priority Order For A Solo Builder

1. docs and architecture
2. onboarding and goal selection
3. adaptive plan calendar
4. session details and manual logging
5. progress
6. social
7. integration adapters

## Practical MVP Cut Lines

If time becomes constrained, keep:

- one strong onboarding flow
- one coherent weekly calendar
- one usable session detail flow
- one clear progress dashboard
- one simple social loop

If needed, reduce:

- number of workout types per category
- number of social surfaces
- depth of progress analytics
- amount of settings and profile detail

## Portfolio Quality Bar

The MVP should show:

- product taste
- technical clarity
- maintainable code boundaries
- thoughtful visual design
- believable roadmap to real integrations
